# Stack Router

Route stack and domain-specific implementation questions. This category is opt-in: use it when the user asks for stack, framework, architecture, or tooling guidance.

## Routing Table

| User intent | Load |
|---|---|
| Choose stack, architecture, libraries, or project defaults | `preferred-stack` |
| Convex schema, query, mutation, action, auth, validation | `convex-patterns` |
| React app structure, routes, state/data boundaries | `frontend-architecture` |
| UI components, styling, color tokens, design system | `ui-component-system` |
| Bun/Turborepo/Convex/Vercel monorepo setup | `monorepo-setup` |
| React component patterns, form sync, `useEffect` risks | `react-patterns` |

## Rule

Do not load stack skills for every coding task. Use workflow skills first for process, then load this router when implementation needs stack-specific guidance.
